# Squarebound · 方格征途

一个致敬童年的战棋闯关游戏：当年在五子棋盘上用黑白棋子加四色角色标识摆出来的战棋梦，
现在用 C++ / raylib 完整实现。

- 15×15 五子棋规格棋盘（含星位）
- 绿子=主角，蓝/红/黄=英雄，白子正面=我方小兵，白子反面=敌方小兵，黑子=障碍
- 英雄血量在面板上以**二进制棋子**展示（当年在棋盘外用正反面棋子记血量的致敬）
- 小兵保留"被击中即阵亡"的原始规则
- 7 个关卡：歼灭 / 夺旗 / 弑君三种目标
- 9 个存档槽位：`Shift+F1~F9` 存档，`F1~F9` 读档
- 附带 CLI 记谱模式与关卡强度自动分析

## 构建与运行

依赖：raylib 5.5+（Arch: `pacman -S raylib`）、支持 C++17 的编译器、系统中任意中文
TTF/OTF 字体（自动通过 `fc-list` 探测，黑体优先；也可用 `SQUAREBOUND_FONT=/path/to/font.ttf`
指定，或放置 `assets/font.ttf`。注意 raylib 无法解析 .ttc 合集）。

```sh
make          # 构建
./squarebound # 图形模式
```

## 玩法

双方轮流行动，每回合可移动**所有**己方单位；每个单位每回合可移动一次并攻击一次。
主角阵亡即败北。

| 操作 | 说明 |
| --- | --- |
| 左键 | 选择单位 → 点绿色格移动 → 点红圈敌人攻击（黄牧师点绿圈友军治疗） |
| 点击自身 | 原地待命 |
| 右键 / Esc | 待命 / 取消选择 |
| 空格 / 结束回合按钮 | 结束回合；敌方回合按住空格加速 |
| Shift+F1~F9 | 存档（我方回合时） |
| F1~F9 | 读档（任意界面） |
| ←/→（标题界面） | 选关 |

## CLI 记谱模式

为 AI 自动对局与关卡分析设计。坐标记谱类似国际象棋：列 `a~o`（左→右），
行 `1~15`（上→下），如 `f13`。

```sh
./squarebound --cli [关卡号]     # 交互 REPL
./squarebound --analyze [局数]   # 全关卡强度分析（AI 双方自走）
./squarebound --bench [局数]     # 全关卡：三种固定原型玩家 vs 当前敌方 AI
./squarebound --test             # 战术回归测试：断言 AI 决策的地面真值
```

### 为什么不能只靠自走

`--analyze` 是**自走**：玩家方也用同一套 `aiActFor`。它的问题是"同源偏差"——
若 AI 有盲点，双方都踩，强度结论会自欺。所以另配了两件独立于自走的工具：

- **`--test`**：一组精确构造的局面 + 期望决策（秒杀、集火脆皮、远程放风筝、群胆压上、
  取价值人头、牧师治疗、魔王范围、夺旗推进）。确定性、不引用自走，是"改动没悄悄踩坏
  旧能力"的回归护栏。写这套测试时它当场揪出了一条我对 AI 行为的错误假设。
- **`--bench`**：玩家方换成**固定不变**的原型策略，作为不随 AI 演化的标尺。单一原型会
  过拟合、且有盲区，故用三种互补原型并取玩家最优原型：
  - **冲锋**：能攻则攻，否则朝最近敌人硬冲（鲁莽）。
  - **风筝**：从最不暴露处射击、打了就跑（惩罚被诱敌/追击的 AI）。
  - **龟缩**：只打送上门的目标、否则固守（专抓"被逐个诱杀"的漏洞，是抗蹲坑回归护栏）。

  三者揭示的东西自走看不到：**敌方 AI 极怕风筝**——初阵自走判 100%、朴素冲锋才 44%，
  可风筝玩家却 100% 碾过；唯有带远程敌巫/魔王的法阵、王座关风筝失灵（48%/1%），
  印证"远程单位是风筝的克星"。龟缩几乎全 0%（超时判负），说明敌方不会被诱敌逐个送死。

| 命令 | 说明 |
| --- | --- |
| `show` / `units` / `sel f13` / `obj` | 查看棋盘 / 单位 / 可达与目标 / 状态 |
| `mv f13 f11` | 移动（放弃攻击） |
| `atk f11 e10` | 原地攻击或治疗 |
| `mva f13 f11 e10` | 移动后攻击或治疗 |
| `wait f13` / `end` | 待命 / 结束回合（敌方行动自动记谱） |
| `save 1` / `load 1` / `level 3` / `restart` | 存读档与跳关（与 GUI 存档互通） |
| `selfplay [局数] [回合上限]` | 内置 AI 双方自走当前关 |
| `analyze [局数]` | 全关卡胜率统计 |

对局记录示例（selfplay 输出）：

```
T2  我 G e11-e8 xe@e5 -4 阵亡
T2  敌 e g5-g6 xs@g8 -2 阵亡
```

### 关卡强度基线（AI 自走 200 局 / 关）

| 关卡 | 敌军数 | 胜率 | 均胜回合 | 主角均余血 | 均存活单位 |
| --- | --- | --- | --- | --- | --- |
| 1 初阵 | 9 | 100% | 8.0 | 9.0 | 1.9 |
| 2 石林 | 12 | 86% | 8.3 | 10.1 | 2.5 |
| 3 突围 | 18 | 97% | 9.3 | 9.2 | 1.9 |
| 4 双将 | 13 | 82% | 7.8 | 8.1 | 2.5 |
| 5 隘口 | 13 | 45% | 10.6 | 4.0 | 1.8 |
| 6 法阵 | 13 | 66% | 7.3 | 6.6 | 3.0 |
| 7 王座 | 11 | 0% | — | — | — |

敌方 AI（一步前瞻：模拟 + 评估）：不再用一堆手调分项常数，而是**模拟每个
「落点 + 攻击目标」在真实规则下的结果，再用一个统一的局面评估函数打分**：
- **评估函数**：己方价值 − 敌方价值（价值 = 单位分值 × 剩余血量比例），己方单位
  站在对方火力下按预期损失折价。击杀、集火、优先脆皮、远程放风筝、避开火力，
  都从这一个评估里自然涌现。
- **出手判定**：进攻若能改善局面（或击杀 / 有同伴壮胆 / 夺旗关）就打；孤立单位
  不做纯亏损的送死换命，被必死火力覆盖时撤退，成群时顶着火力压上。

这个基线是 AI 对 AI（自走"玩家"不会用诱饵、间距、集火等人类战术），人类玩家的
实际胜率会更高——上表数值越低表示越吃操作。王座关是终局，自走玩家打不过属正常
（经人工 CLI 实测可胜）。改关卡只需编辑 `src/main.cpp` 的 ASCII 地图，再跑
`./squarebound --analyze 200` 对比本表。

## 代码

单文件 `src/main.cpp`（LLVM 风格，2 空格缩进）。关卡用 ASCII 地图定义
（`X` 黑子、`G/B/R/Y/s` 我方、`e/c/m/K` 敌方、`h` 药草、`E` 旗帜），
改地图即可调关卡。存档为纯文本 `saves/slotN.txt`。
