# Squarebound · 方格征途

一个致敬童年的战棋闯关游戏：当年在五子棋盘上用黑白棋子加四色角色标识摆出来的战棋梦，
现在用 C++ / raylib 完整实现。

- 15×15 五子棋规格棋盘（含星位）
- 绿子=主角，蓝/红/黄=英雄，白子正面=我方小兵，白子反面=敌方小兵，黑子=障碍
- 英雄血量在面板上以**二进制棋子**展示（当年在棋盘外用正反面棋子记血量的致敬）
- 小兵保留"被击中即阵亡"的原始规则
- 7 个关卡：歼灭 / 夺旗 / 弑君三种目标
- 9 个存档槽位：`Shift+F1~F9` 存档，`F1~F9` 读档
- 附带 CLI 记谱模式与关卡强度自动分析

## 构建与运行

依赖：raylib 5.5+（Arch: `pacman -S raylib`）、支持 C++17 的编译器、系统中任意中文
TTF/OTF 字体（自动通过 `fc-list` 探测，黑体优先；也可用 `SQUAREBOUND_FONT=/path/to/font.ttf`
指定，或放置 `assets/font.ttf`。注意 raylib 无法解析 .ttc 合集）。

```sh
make          # 构建
./squarebound # 图形模式
```

## 玩法

双方轮流行动，每回合可移动**所有**己方单位；每个单位每回合可移动一次并攻击一次。
主角阵亡即败北。

| 操作 | 说明 |
| --- | --- |
| 左键 | 选择单位 → 点绿色格移动 → 点红圈敌人攻击（黄牧师点绿圈友军治疗） |
| 点击自身 | 原地待命 |
| 右键 / Esc | 待命 / 取消选择 |
| 空格 / 结束回合按钮 | 结束回合；敌方回合按住空格加速 |
| Shift+F1~F9 | 存档（我方回合时） |
| F1~F9 | 读档（任意界面） |
| ←/→（标题界面） | 选关 |

### 兵种

| 棋子 | 名称 | 血量 | 攻击 | 防御 | 射程 | 移动 | 备注 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 绿 | 主角·远征者 | 12 | 4 | 1 | 3 | 3 | 阵亡即败北 |
| 蓝 | 蓝骑士 | 14 | 5 | 2 | 1 | 4 | 重甲近战先锋 |
| 红 | 红法师 | 8 | 6 | 0 | 3 | 2 | 高伤脆皮 |
| 黄 | 黄牧师 | 9 | 2 | 0 | 2 | 3 | 治疗友军 +4 |
| 白(正) | 义勇兵 | 1 | 2 | 0 | 2 | 2 | 触死 |
| 白(反) | 敌方小兵 | 1 | 2 | 0 | 2 | 2 | 触死 |
| 黑(红圈) | 敌将 | 9 | 4 | 1 | 1 | 3 | 需集火 2~3 刀的精英 |
| 黑(紫圈) | 敌巫 | 6 | 5 | 0 | 3 | 2 | |
| 黑(金圈) | 魔王 | 20 | 5 | 2 | 2 | 2 | 范围攻击：波及射程内所有我方单位 |

规则细节：**实际伤害 = 攻击 − 防御，保底 1 点**（`MinDmg` 常量可调）；远程攻击
需要视线，黑子会挡住弹道（近战不受影响）；友军可穿行但不可停留在同一格；踩到
药草回复 4 点血量。

## CLI 记谱模式

为 AI 自动对局与关卡分析设计。坐标记谱类似国际象棋：列 `a~o`（左→右），
行 `1~15`（上→下），如 `f13`。

```sh
./squarebound --cli [关卡号]   # 交互 REPL
./squarebound --analyze [局数] # 全关卡强度分析
```

| 命令 | 说明 |
| --- | --- |
| `show` / `units` / `sel f13` / `obj` | 查看棋盘 / 单位 / 可达与目标 / 状态 |
| `mv f13 f11` | 移动（放弃攻击） |
| `atk f11 e10` | 原地攻击或治疗 |
| `mva f13 f11 e10` | 移动后攻击或治疗 |
| `wait f13` / `end` | 待命 / 结束回合（敌方行动自动记谱） |
| `save 1` / `load 1` / `level 3` / `restart` | 存读档与跳关（与 GUI 存档互通） |
| `selfplay [局数] [回合上限]` | 内置 AI 双方自走当前关 |
| `analyze [局数]` | 全关卡胜率统计 |

对局记录示例（selfplay 输出）：

```
T2  我 G e11-e8 xe@e5 -4 阵亡
T2  敌 e g5-g6 xs@g8 -2 阵亡
```

### 关卡强度基线（AI 自走 50 局 / 关，防御系统版本）

| 关卡 | 胜率 | 均胜回合 | 主角均余血 | 均存活单位 |
| --- | --- | --- | --- | --- |
| 1 初阵 | 50/50 | 5.5 | 11.3 | 2.0 |
| 2 石林 | 50/50 | 6.3 | 11.6 | 2.5 |
| 3 突围 | 50/50 | 9.0 | 12.0 | 4.7 |
| 4 双将 | 49/50 | 7.5 | 10.4 | 2.7 |
| 5 隘口 | 50/50 | 5.4 | 10.9 | 5.2 |
| 6 法阵 | 46/50 | 7.1 | 9.6 | 2.6 |
| 7 王座 | 2/50 | 9.0 | 2.0 | 2.5 |

敌方 AI 机制（收益/风险决策 + 群胆）：
- **收益打分**：实际伤害×15、击杀 +1000、补刀残血、优先脆皮、远程保持最大射程、
  魔王按波及人数加权，外加**群攻协同**（能一起够到同一目标的友军每个 +15）。
- **风险打分**：落点处我方"移动+射程"威胁伤害 ×2；威胁 ≥ 自身血量的落点追加
  必死惩罚。净亏损的换命在有退路时会被放弃。
- **机动与群胆**：够不着目标时在我方火力圈外集结推进；被火力覆盖时，附近
  （4 格内）有 2 个以上同伴就**顶着火力压上接敌**，只有孤立且必死的单位才
  撤到圈边；夺旗关敌人始终全力拦截。

因此：蹲坑钓鱼钓不到（敌人在圈外列阵）、孤兵不送死（拒绝亏本换命）、但敌人
成群时敢于冒险强攻——你的单位进入敌群合围范围会被**连续围殴**（实测主角孤军
深入两回合内被打掉 8 血）。为配合更敢打的 AI，各关敌兵数量均有增加。上表是
AI 对 AI 基线（自走"玩家"不会用诱饵和间距，王座关打不过很正常，该关经人工
CLI 实测可胜）。

## 代码

单文件 `src/main.cpp`（LLVM 风格，2 空格缩进）。关卡用 ASCII 地图定义
（`X` 黑子、`G/B/R/Y/s` 我方、`e/c/m/K` 敌方、`h` 药草、`E` 旗帜），
改地图即可调关卡。存档为纯文本 `saves/slotN.txt`。
